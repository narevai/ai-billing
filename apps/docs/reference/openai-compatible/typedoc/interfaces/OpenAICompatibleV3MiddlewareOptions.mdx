# Interface: OpenAICompatibleV3MiddlewareOptions\<TTags\>

Defined in: [openai-compatible/src/ai-sdk/language-model-middleware/v3/language-model-v3-openai-compatible-billing-middleware.ts:29](https://github.com/narevai/ai-billing/blob/main/packages/openai-compatible/src/ai-sdk/language-model-middleware/v3/language-model-v3-openai-compatible-billing-middleware.ts#L29)

Configuration for [createOpenAICompatibleV3Middleware](../functions/createOpenAICompatibleV3Middleware).

Extends BaseBillingMiddlewareOptions (`destinations`, `defaultTags`, `waitUntil`, `onError`) and
adds `priceResolver` plus `providerId`. Usage comes from the AI SDK's normalized usage fields for
OpenAI-compatible responses; cost is computed from that usage and the resolved ModelPricing using
the same rules as the package's cost helper.

`providerId` must match the `name` you pass to `createOpenAICompatible` from `@ai-sdk/openai-compatible`
so PriceResolver calls and emitted `provider` on events stay aligned.
